⚠️ Challenges Encountered

Real Obstacles,
Real Solutions.

Every challenge was identified, diagnosed, and resolved — no blockers remain for M2.

🐍
Python 3.14 Incompatibility
Resolved ✓
Problem: psycopg2-binary had no wheel for Python 3.14, breaking the install.
Fix: Migrated to psycopg3 (psycopg[binary]) which fully supports Python 3.14. Zero functionality loss.
🐳
Docker Desktop Not Running
Workaround Active
Problem: Docker Desktop failed to start on Windows, blocking local PostgreSQL + Redis.
Workaround: Using SQLite in-memory for Django testing settings. Full Docker stack validated in compose file. Resolution: ensure Docker Desktop service is running before M2.
📦
WeasyPrint Windows Incompatibility
Resolved ✓
Problem: WeasyPrint (PDF receipts) requires GTK libraries — unavailable on Windows dev machine.
Fix: Replaced with xhtml2pdf which is pure-Python and cross-platform. PDF receipt quality maintained.
🔌
DISCO API Access Pending
Open — Business
Problem: No DISCO sandbox API credentials yet. Blocks M2 real-world meter validation testing.
Mitigation: Mock DISCO API server to be built in M2 so backend development is not blocked while partnerships are finalized.
💳
Gateway Merchant Accounts
Open — Finance
Problem: Paystack and Flutterwave merchant accounts not yet approved for live transactions.
Mitigation: Both gateways provide sandbox/test mode — M3 payment integration will proceed fully with test keys.
⚖️
CBN Regulatory Clearance
Parallel Track
Problem: CBN intermediary authorization is a prerequisite for live financial transactions.
Plan: Legal team pursuing CBN authorization in parallel with development. Target: clearance before M7 soft-launch gate.
